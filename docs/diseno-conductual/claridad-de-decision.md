# Claridad de decisión

La claridad de decisión es el primer pilar conductual de Contextic.

Una página puede estar visualmente cuidada y aun así fallar si el usuario tiene que descifrar qué importa. Contextic debería ayudar a detectar cuándo la interfaz obliga al usuario a comparar, dudar o adivinar.

## Qué debería buscar Contextic

### Prioridad de acción

Preguntas:

- ¿Hay una acción dominante por bloque de decisión?
- ¿Las acciones secundarias tienen menor prioridad visual?
- ¿La acción destructiva está separada de la acción constructiva?

Señales:

- múltiples elementos tipo botón con fondos fuertes,
- tamaño y contraste similares entre CTAs no equivalentes,
- CTAs agrupados sin jerarquía.

### Claridad de copy

Preguntas:

- ¿El copy del CTA describe el resultado?
- ¿El usuario puede anticipar qué ocurrirá después?
- ¿Se usa jerga donde un lenguaje más claro reduciría esfuerzo?

Señales:

- etiquetas genéricas,
- mismo copy de CTA con destinos diferentes,
- etiquetas de acción separadas del contenido que las justifica.

### Estructura de elección

Preguntas:

- ¿Las opciones están agrupadas de forma significativa?
- ¿Existe una opción recomendada o predeterminada cuando procede?
- ¿Se muestran demasiadas opciones equivalentes a la vez?

Señales:

- grandes grupos de tarjetas o botones equivalentes,
- ausencia de opción destacada o predeterminada,
- zonas de comparación densas sin revelado progresivo.

## Estilo de salida

Salida débil:

> Esta página tiene demasiados botones.

Mejor salida para Contextic:

> Se detectan 3 acciones visualmente prominentes en el primer viewport. Esto puede debilitar la claridad de decisión porque el usuario debe inferir qué acción es más importante. Considera mantener una única acción primaria en este bloque y degradar las alternativas a variantes secundarias o enlaces de texto.
